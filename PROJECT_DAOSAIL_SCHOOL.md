PROJECT_DAOSAIL_SCHOOL.md (v.1.1)

1. Главная цель проекта

Создание интерактивной онлайн-школы "Школа крейсерского яхтинга". Главная особенность школы — персональный ИИ-инструктор, который помогает каждому ученику осваивать материал в своем темпе.

2. Наш фокус на первом этапе (MVP - Минимально Жизнеспособный Продукт)

Задача: Запустить работающую платформу с одним полноценным курсом "Введение в яхтинг". На этом этапе мы не занимаемся "клубом", "походами" или "DAO". Наша цель — создать качественный образовательный продукт.

3. Для кого мы это делаем (Целевая аудитория)

Абсолютные новички без опыта в яхтинге.

Люди с минимальным опытом, которые хотят систематизировать свои знания.

Те, кто интересуется яхтингом, но не имеет доступа к обычным школам.

4. Из чего будет состоять наше приложение (Ключевые функции MVP)

4.1. Система для учеников:

Регистрация и профиль: Пользователь может создать аккаунт, указать свое имя и уровень опыта.

Личный кабинет: Место, где ученик видит свой прогресс, пройденные уроки и достижения (бейджи).

База знаний: Простой справочник с поиском по яхтенным терминам и основным схемам.

4.2. Обучающий процесс:

Основной курс: Курс "Введение в яхтинг", состоящий из 7 уроков.

Структура урока: Каждый урок включает текст, иллюстрации, короткие видео (анимации), интерактивные схемы и проверочный тест.

Система тестов: После каждого урока — тест. Для получения сертификата нужно сдать итоговый тест.

ИИ-Инструктор: В каждом уроке есть чат, где можно задать вопрос ИИ-помощнику по теме урока.

5. Содержание нашего первого курса ("Введение в яхтинг")

Курс состоит из 7 ключевых тем:

Урок 1: История и основные термины.

Урок 2: Устройство парусной яхты.

Урок 3: Как яхта движется под парусом.

Урок 4: Основы управления яхтой.

Урок 5: Базовые навигационные знания.

Урок 6: Правила безопасности на воде.

Урок 7: Планирование первого выхода на воду.

6. На чем мы это строим (Технологический фундамент)

6.1. Основа проекта

Мы не начинаем с нуля. В качестве фундамента для Школы используется архитектура существующего прототипа приложения DAOsail (согласно документу PROJECT_ARCHITECTURE.md). Это дает нам огромное преимущество и готовые модули:

Frontend: Next.js + React + TypeScript.

UI Kit: Tailwind CSS + Radix UI.

Backend и База данных: Supabase (PostgreSQL).

Аутентификация: Уже настроенная система регистрации и входа через Supabase Auth.

Система профилей: Готовый ProfileService с ролями и статистикой.

ИИ-интеграция: Рабочий ChatService и EmbeddingService для RAG-системы.

6.2. План адаптации существующей архитектуры

Наша основная работа будет заключаться в адаптации и расширении этого фундамента под нужды Школы:

База данных: Расширить схему Supabase, добавив таблицы Courses, Lessons, Tests, UserProgress.

Интерфейс (UI):

Адаптировать навигацию: убрать разделы "DAO", "Сообщества" и добавить "Курсы", "Библиотека".

Переработать главную страницу в лендинг для Школы.

Создать интерфейсы для отображения уроков и прохождения тестов.

Сервисы (Логика):

Модифицировать ProfileService для отслеживания прогресса по курсам и урокам.

Создать новые сервисы для управления учебным контентом.
